# Исправление множественного открытия диалогов

## Проблема

В игре Elemental Blast наблюдалась критическая проблема: диалоговые окна открывались по несколько штук при следующих событиях:
- Победа на уровне
- Поражение на уровне  
- Нажатие на кнопку старта уровня
- Покупка бустеров

Это приводило к:
- Зависанию игры
- Множественному отображению одинаковых окон
- Невозможности корректно закрыть диалоги

## Причины

### 1. Отсутствие защиты в `_process()`

В `game_board.gd` функции `_on_level_completed()` и `_on_level_failed()` вызывались внутри `_process()`, который выполняется **каждый кадр** (60 раз в секунду).

```gdscript
# ДО ИСПРАВЛЕНИЯ (ПЛОХО)
func _process(delta: float) -> void:
    # ...
    if _projectiles.is_empty() and _active_anims.is_empty() and _enemy_death_anims.is_empty():
        if _check_level_completed():
            _on_level_completed()  # ← Вызывается каждый кадр!
        elif _moves_left == 0 or _player_lives == 0:
            _on_level_failed()     # ← Вызывается каждый кадр!
```

**Результат**: Если уровень завершён и все анимации закончились, функции победы/поражения вызываются 60 раз в секунду → диалоги создаются снова и снова.

### 2. Отсутствие защиты от множественных кликов

В `level_start_dialog.gd` кнопки "Играть" и крестик не защищены от быстрых повторных нажатий:

```gdscript
# ДО ИСПРАВЛЕНИЯ (ПЛОХО)
play_btn.pressed.connect(func():
    if not is_queued_for_deletion():
        _on_start_pressed()  # ← Может вызваться несколько раз при быстрых кликах
)
```

### 3. Отсутствие защиты от повторного открытия

В `main_menu.gd` функция `_show_level_start_dialog()` не проверяла, открыт ли уже диалог:

```gdscript
# ДО ИСПРАВЛЕНИЯ (ПЛОХО)
func _show_level_start_dialog(level: int):
    var dialog = Control.new()
    # ...
    add_child(dialog)  # ← Создаёт новый диалог при каждом вызове
```

## Решение

### 1. Флаги однократного срабатывания в `game_board.gd`

Добавлены булевы флаги для отслеживания уже произошедших событий:

```gdscript
# Флаги для предотвращения множественного срабатывания диалогов
var _level_completed_triggered: bool = false
var _level_failed_triggered: bool = false
```

Проверка перед вызовом:

```gdscript
# ПОСЛЕ ИСПРАВЛЕНИЯ (ХОРОШО)
func _process(delta: float) -> void:
    # ...
    if _projectiles.is_empty() and _active_anims.is_empty() and _enemy_death_anims.is_empty():
        if _check_level_completed() and not _level_completed_triggered:
            _level_completed_triggered = true  # ← Блокируем повторный вызов
            _on_level_completed()
        elif (_moves_left == 0 or _player_lives == 0) and not _level_failed_triggered:
            _level_failed_triggered = true     # ← Блокируем повторный вызов
            _on_level_failed()
```

**Результат**: Функции победы/поражения вызываются **ровно один раз** за всю игровую сессию.

### 2. Флаг закрытия диалога в `level_start_dialog.gd`

Добавлен флаг для отслеживания процесса закрытия диалога:

```gdscript
# Флаг для предотвращения множественного закрытия диалога
var _dialog_closing: bool = false
```

Защита кнопок:

```gdscript
# ПОСЛЕ ИСПРАВЛЕНИЯ (ХОРОШО)
play_btn.pressed.connect(func():
    if not _dialog_closing and not is_queued_for_deletion():
        _dialog_closing = true  # ← Блокируем повторные нажатия
        _on_start_pressed()
)

close_btn.pressed.connect(func():
    if not _dialog_closing and not is_queued_for_deletion():
        _dialog_closing = true  # ← Блокируем повторные нажатия
        _on_start_pressed()
)
```

**Результат**: Даже при быстрых многократных кликах диалог закрывается **только один раз**.

### 3. Флаг открытого диалога в `main_menu.gd`

Добавлен флаг для отслеживания открытого диалога старта уровня:

```gdscript
# Флаг для предотвращения множественного открытия диалога
var _level_start_dialog_shown: bool = false
```

Защита от повторного открытия:

```gdscript
# ПОСЛЕ ИСПРАВЛЕНИЯ (ХОРОШО)
func _show_level_start_dialog(level: int):
    # Предотвращаем множественное открытие диалога
    if _level_start_dialog_shown:
        return  # ← Прерываем, если диалог уже открыт
    
    _level_start_dialog_shown = true
    
    var dialog = Control.new()
    # ...
    
    # Сбрасываем флаг при удалении диалога
    dialog.tree_exiting.connect(func():
        _level_start_dialog_shown = false  # ← Разрешаем открытие нового диалога
    )
    
    add_child(dialog)
```

**Результат**: Диалог старта уровня открывается **максимум один раз** одновременно. После закрытия флаг сбрасывается, позволяя открыть диалог снова.

## Паттерн решения

Универсальный паттерн для предотвращения множественного срабатывания событий:

```gdscript
# 1. Объявляем флаг
var _action_triggered: bool = false

# 2. Проверяем флаг перед действием
func trigger_action():
    if _action_triggered:
        return  # Уже выполнено, выходим
    
    _action_triggered = true  # Блокируем повторное выполнение
    
    # Выполняем действие
    do_something()
    
    # Опционально: сбрасываем флаг при удалении/завершении
    cleanup_signal.connect(func():
        _action_triggered = false
    )
```

## Проверка исправления

### Тестовые сценарии

1. **Диалог старта уровня**
   - Нажать на кнопку уровня в главном меню
   - Ожидаемый результат: диалог появляется **один раз**
   - Быстро кликнуть несколько раз подряд
   - Ожидаемый результат: диалог всё равно **один**

2. **Диалог победы**
   - Пройти уровень (уничтожить всех монстров)
   - Ожидаемый результат: диалог победы появляется **один раз**

3. **Диалог поражения**
   - Потратить все ходы или жизни
   - Ожидаемый результат: диалог поражения появляется **один раз**

4. **Кнопки в диалогах**
   - Открыть любой диалог
   - Быстро кликнуть на кнопку "Играть"/"Закрыть" несколько раз
   - Ожидаемый результат: действие выполняется **только один раз**

### Ожидаемое поведение после исправления

✅ Все диалоги открываются **строго один раз**  
✅ Повторные клики не вызывают дубликатов  
✅ Игра не зависает от множественных диалогов  
✅ Флаги корректно сбрасываются после завершения событий  

## Заключение

Проблема множественных диалогов была вызвана отсутствием **механизма защиты от повторного срабатывания** в критических местах кода. 

Решение основано на использовании **булевых флагов**, которые:
1. Устанавливаются при первом срабатывании события
2. Проверяются перед выполнением действия
3. Сбрасываются при необходимости (для повторного использования)

Этот подход обеспечивает **строгий контроль** над жизненным циклом диалогов и предотвращает все случаи их дублирования.
