# Система бустеров

## Описание

Постоянные усиления, которые игрок может использовать для упрощения прохождения уровней. Бустеры сохраняются между уровнями и могут быть куплены за монеты.

## Типы бустеров

### 1. Молоток (HAMMER)
- **Действие**: Уничтожает одну фишку
- **Область**: Зона игрока
- **Требует**: Выбор цели (клик по фишке)

### 2. Ракета (ROW_BLAST)
- **Действие**: Уничтожает весь ряд фишек
- **Область**: Зона игрока
- **Требует**: Выбор цели (клик по любой фишке в ряду)

### 3. Перемешивание (SHUFFLE)
- **Действие**: Перемешивает все фишки на поле
- **Область**: Вся зона игрока
- **Требует**: Нет (мгновенное применение)

### 4. Заморозка (FREEZE)
- **Действие**: Останавливает монстров на 1 ход
- **Область**: Зона врагов
- **Требует**: Клик по зоне врагов

## Константы

```gdscript
const INITIAL_BOOSTERS := 4        # Начальное количество каждого бустера
const BOOSTER_PURCHASE_COST := 350 # Цена покупки 1 заряда

enum BoosterType {
    HAMMER = 1,
    ROW_BLAST = 2,
    SHUFFLE = 3,
    FREEZE = 4
}
```

### Функции управления

```gdscript
func use_booster(type: BoosterType) -> bool:
    # Использовать бустер (уменьшить количество)
    if booster_counts.get(type, 0) > 0:
        booster_counts[type] -= 1
        _save_progress()
        emit_signal("boosters_changed")
        return true
    return false

func buy_booster(type: BoosterType) -> bool:
    # Купить бустер за монеты
    if spend_coins(BOOSTER_PURCHASE_COST):
        booster_counts[type] = booster_counts.get(type, 0) + 1
        _save_progress()
        emit_signal("boosters_changed")
        return true
    return false

func get_booster_count(type: BoosterType) -> int:
    # Получить количество бустеров
    return booster_counts.get(type, 0)
```

## Игровой процесс

### Использование бустера

```
1. Игрок кликает на кнопку бустера
   ↓
2. Проверка: есть ли заряды?
   ├─ Да → Активация бустера
   └─ Нет → Диалог покупки
```

### Активация бустера (с целью)

```

   ↓
Бустер подсвечивается
   ↓
Игрок кликает по цели на поле
   ↓

   ↓
Количество уменьшается на 1
   ↓

```

### Активация мгновенного бустера

```

   ↓
Применяется сразу
   ↓
Количество уменьшается на 1
   ↓

```

### Покупка бустера

```
Клик на бустер без зарядов
   ↓
Диалог покупки:
  - Название бустера
  - Цена: 350 монет
  - Баланс игрока
   ↓
Игрок подтверждает
   ↓
Проверка монет (>= 350?)
   ├─ Да → Списание 350 монет
   │       Добавление +1 заряда
   │       Сохранение
   └─ Нет → Кнопка "Купить" неактивна
```

```gdscript
func _show_buy_booster_dialog(lm_type: int):
    var booster_names = {
        LevelManager.BoosterType.HAMMER: "Молоток",
        LevelManager.BoosterType.ROW_BLAST: "Ракета",
        LevelManager.BoosterType.SHUFFLE: "Перемешивание",
        LevelManager.BoosterType.FREEZE: "Заморозка"
    }
    
    var booster_name = booster_names.get(lm_type, "Бустер")
    var cost = LevelManager.BOOSTER_PURCHASE_COST
    var player_coins = LevelManager.get_coins()
    
    var dialog = AcceptDialog.new()
    dialog.title = "Купить бустер?"
    dialog.dialog_text = "Бустер: %s\nЦена: %d монет\n\nУ вас: %d монет" % [booster_name, cost, player_coins]
    dialog.ok_button_text = "Купить (%d)" % cost
    dialog.cancel_button_text = "Отмена"
    
    if player_coins < cost:
        dialog.dialog_text = "Недостаточно монет!\n\nБустер: %s\nЦена: %d монет\nУ вас: %d монет" % [booster_name, cost, player_coins]
        dialog.ok_button_text = "Понятно"
        dialog.get_ok_button().disabled = true
```

## Маппинг типов

Между `game_board.gd` и `LevelManager.gd` используются разные enum, поэтому нужна конвертация:

```gdscript
func _convert_to_lm_booster_type(type: BoosterType) -> int:
    match type:
        BoosterType.HAMMER: return LevelManager.BoosterType.HAMMER
        BoosterType.ROW_BLAST: return LevelManager.BoosterType.ROW_BLAST
        BoosterType.SHUFFLE: return LevelManager.BoosterType.SHUFFLE
        BoosterType.FREEZE: return LevelManager.BoosterType.FREEZE
    return LevelManager.BoosterType.HAMMER
```

## Экономика бустеров

### Стоимость использования

Прямой стоимости нет - бустеры бесплатны в использовании, если есть заряды.

### Стоимость покупки

- **Цена**: 350 монет за 1 заряд
- **Фиксированная**: не зависит от уровня или количества покупок

## Баланс

### Рекомендуемое соотношение

**Награда за уровень**: 50-200 монет  
**Цена бустера**: 350 монет  

**Вывод**: Нужно пройти 2-7 уровней для покупки 1 бустера.

### Стратегии игрока

1. **Накопление**:
   - Не использовать бустеры на простых уровнях
   - Накопить монеты и бустеры для сложных уровней

2. **Равновесие**:
   - Использовать бустеры по необходимости
   - Покупать только при критической нехватке

3. **Агрессивная**:
   - Активно использовать бустеры
   - Часто покупать для комфортного прохождения
   - Требует хорошего дохода монет

## UI отображение

### Кнопка бустера

```
┌─────────────┐
│   [Иконка]  │
│             │
│      4  ◄───┼─ Счётчик зарядов
└─────────────┘
```

### Состояния кнопки

1. **Есть заряды** (count > 0):
   - Кнопка активна
   - Счётчик показывает количество
   - Клик → активация или применение

2. **Нет зарядов** (count = 0):
   - Кнопка активна (!)
   - Счётчик показывает 0
   - Клик → диалог покупки

3. **Активирована** (для бустеров с целью):
   - Подсветка: Color(1.5, 1.5, 1.0)
   - Ожидание выбора цели
   - Повторный клик → деактивация

- Покупке бустера
- Загрузке сохранения

## Особенности реализации

### Постоянство

Бустеры **НЕ** сбрасываются при:
- Переходе на новый уровень
- Победе/поражении
- Выходе из игры
- Перезапуске приложения

Бустеры сбрасываются только при:
- Полном удалении сохранения (вручную)
- Новой игре (если реализовано)

### Независимость от уровня

Количество бустеров одинаково на всех уровнях:
- Нет автоматического пополнения
- Нет бонусов за прохождение
- Единственный способ получить - купить

### Приоритет покупки

При нехватке монет:
- Кнопка "Купить" деактивируется
- Диалог остаётся открытым
- Игрок может закрыть диалог
- Бустер не применяется

## Тестирование

### Тестовые сценарии

1. **Использование бустера**:
   - Начальное количество: 4
   - Использовать → Ожидание: 3
   - Перезагрузить игру → Ожидание: 3 (сохранено)

2. **Покупка бустера**:
   - Монеты: 500
   - Купить молоток → 150 монет, молоток +1
   - Купить ракету → -200 монет (недостаточно для второй)

3. **Нехватка монет**:
   - Монеты: 100
   - Клик на бустер с 0 зарядов
   - Диалог с неактивной кнопкой

4. **Сохранение между уровнями**:
   - Уровень 1: использовать 2 бустера
   - Перейти на уровень 2
   - Проверить количество (должно остаться то же)

### Команды отладки

```gdscript
# Добавить бустеры
for type in [1, 2, 3, 4]:
    LevelManager.booster_counts[type] += 5

# Обнулить бустеры
for type in [1, 2, 3, 4]:
    LevelManager.booster_counts[type] = 0

# Проверить количество
print("Boosters:", LevelManager.booster_counts)
```

## Заключение

Система бустеров обеспечивает:

✅ **Постоянство**: Сохранение между уровнями  
✅ **Экономику**: Покупка за монеты  
✅ **Стратегию**: Выбор когда использовать  
✅ **Прогрессию**: Накопление ресурсов  
✅ **Гибкость**: Различные типы бустеров  

Игрок получает стартовый набор (4 каждого) и сам решает, когда их использовать и покупать ли новые.
