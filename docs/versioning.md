# Система версионирования проекта Elemental Blast

## Обзор

Проект использует автоматическую систему версионирования на основе git-веток.

## Правила версионирования

### Основная ветка (main)
- Версия хранится в файле `VERSION` в корне проекта
- Текущая версия: **0.1**
- При слиянии feature-ветки в main версия увеличивается на 0.1 (например, 0.1 → 0.2)

### Feature-ветки
- Версия формируется автоматически: `[версия main].[последние 5 символов имени ветки]`
- Пример: для ветки `cursor/-bc-191fe751-465f-5e5c-8abe-046092005dd5-559b`
  - Базовая версия main: 0.1
  - Последние 5 символов: 559b
  - Итоговая версия: **0.1.559b**

## Компоненты системы

### 1. Файл VERSION
```
/workspace/VERSION
```
Содержит базовую версию для ветки main (например, `0.1`)

### 2. Скрипт get_version.sh
```bash
/workspace/get_version.sh
```
Bash-скрипт, автоматически определяющий версию на основе текущей git-ветки.

### 3. VersionManager.gd
```
/workspace/scripts/VersionManager.gd
```
Autoload singleton в Godot, который:
- Загружает версию при запуске игры
- Определяет текущую git-ветку
- Предоставляет API для получения информации о версии

## API VersionManager

### Свойства
- `version: String` - текущая версия проекта
- `branch: String` - текущая git-ветка

### Методы
- `get_version() -> String` - возвращает версию (например, "0.1.559b")
- `get_branch() -> String` - возвращает имя ветки
- `get_full_version_string() -> String` - возвращает полную строку версии (например, "v0.1.559b (cursor/...)")

## Использование в коде

### Получение версии в GDScript
```gdscript
# Получить версию
var current_version = VersionManager.get_version()
print("Текущая версия: ", current_version)

# Получить ветку
var current_branch = VersionManager.get_branch()
print("Текущая ветка: ", current_branch)

# Получить полную строку версии
var full_version = VersionManager.get_full_version_string()
print(full_version)  # v0.1.559b (cursor/-bc-...-559b)
```

### Отображение версии в UI
```gdscript
# В меню или экране настроек
$VersionLabel.text = "Версия " + VersionManager.get_version()
```

## Обновление версии при слиянии

### При слиянии feature-ветки в main:

1. Откройте файл `VERSION`
2. Увеличьте версию на 0.1:
   ```
   0.1 → 0.2
   0.2 → 0.3
   и т.д.
   ```
3. Зафиксируйте изменение:
   ```bash
   git add VERSION
   git commit -m "Bump version to 0.2"
   ```

## Проверка версии

### В консоли через скрипт
```bash
./get_version.sh
```

### При запуске Godot проекта
Версия автоматически выводится в консоль при старте:
```
Elemental Blast версия: 0.1.559b
Git ветка: cursor/-bc-191fe751-465f-5e5c-8abe-046092005dd5-559b
```

## Troubleshooting

### Версия не определяется автоматически
- Убедитесь, что `get_version.sh` имеет права на выполнение: `chmod +x get_version.sh`
- Проверьте, что git доступен в системе: `which git`
- Проверьте, что файл `VERSION` существует и содержит корректную версию

### Версия всегда показывает 0.1
- Убедитесь, что VersionManager правильно загружается как autoload
- Проверьте логи Godot на наличие ошибок при выполнении скрипта
