# Исправление бага отображения `<null>` после слияния PR версионирования

## Дата
23 марта 2026

## Симптомы проблемы

После слияния PR #12 (система версионирования) игра перестала корректно работать:

1. **Не отображались уровни** - сетка уровней была пустой
2. **`<null>` вместо золота** - количество монет не отображалось
3. **`<null>` в кнопке уровня** - текст кнопки показывал "Уровень `<null>`"
4. **`<null>` у бустеров** - количество бустеров не отображалось
5. **Проблемы с диалогом начала уровня** - диалог не показывал корректные данные

## Причины

### 1. Дублирование переменной `player_coins` в LevelManager.gd
```gdscript
# НЕПРАВИЛЬНО (было два раза):
var player_coins: int = INITIAL_COINS  # Строка 13
var player_coins: int = 500            # Строка 23
```

Это приводило к неопределённому поведению и потенциальному значению `null`.

### 2. Неправильный доступ к словарям в `_save_progress()` и `_load_progress()`
```gdscript
# НЕПРАВИЛЬНО:
cfg.set_value("prelevel_boosts", "bomb", prelevel_boosts.bomb)

# ПРАВИЛЬНО:
cfg.set_value("prelevel_boosts", "bomb", prelevel_boosts["bomb"])
```

GDScript требует использования квадратных скобок `[]` для доступа к элементам словаря.

### 3. Дублирование функций управления монетами
Функции `add_coins()`, `spend_coins()`, `get_coins()` были объявлены дважды:
- Первый раз (строки 160-172) - без эмита сигнала `coins_changed`
- Второй раз (строки 235-249) - с эмитом сигнала

Это приводило к непредсказуемому поведению.

### 4. Проблемы с VersionManager
`VersionManager.gd` пытался выполнить bash-скрипт через `res://`, что недоступно во время выполнения игры.

## Решение

### 1. LevelManager.gd
- ✅ Удалено дублирование `player_coins`
- ✅ Исправлен синтаксис доступа к словарям (`prelevel_boosts["key"]`)
- ✅ Удалены дублирующиеся функции управления монетами
- ✅ Добавлено логирование инициализации

### 2. VersionManager.gd
- ✅ Упрощена логика загрузки версии
- ✅ Теперь читается только файл `VERSION` + проверка git-ветки
- ✅ Убрана зависимость от внешнего bash-скрипта

### 3. main_menu.gd
- ✅ Добавлены проверки `if LevelManager` перед использованием
- ✅ Защита от null в функциях отображения UI

### 4. level_start_dialog.gd
- ✅ Добавлены проверки `if LevelManager` перед использованием
- ✅ Защита от null при получении данных

## Код исправлений

### До (неправильно):
```gdscript
# LevelManager.gd
var player_coins: int = INITIAL_COINS
# ... 10 строк ...
var player_coins: int = 500  # ДУБЛИКАТ!

func _save_progress():
    cfg.set_value("prelevel_boosts", "bomb", prelevel_boosts.bomb)  # ОШИБКА!
```

### После (правильно):
```gdscript
# LevelManager.gd
var player_coins: int = INITIAL_COINS  # Только одно объявление

func _save_progress():
    cfg.set_value("prelevel_boosts", "bomb", prelevel_boosts["bomb"])  # Правильно
```

### Защита от null в UI:
```gdscript
# main_menu.gd
func _update_level_label():
    if is_instance_valid(play_button):
        var lvl = LevelManager.current_level if LevelManager else 1
        play_button.text = "Уровень " + str(lvl)
```

## Урок на будущее

1. **Не дублируйте переменные** - используйте уникальные имена или проверяйте код перед коммитом
2. **Используйте правильный синтаксис GDScript** - для словарей всегда `dict["key"]`, не `dict.key`
3. **Не дублируйте функции** - используйте поиск по коду перед добавлением новых функций
4. **Добавляйте защиту от null** - особенно при работе с autoload синглтонами
5. **Тестируйте после слияния PR** - запускайте игру и проверяйте основной функционал
