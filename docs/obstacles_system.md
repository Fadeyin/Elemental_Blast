# Система препятствий (стены/башни)

## Описание

Статичные объекты на игровом поле, которые имеют здоровье, блокируют движение монстров и могут быть разрушены снарядами игрока.

## Основные характеристики

### Визуал
- **Форма**: Прямоугольник, занимающий полную клетку
- **Цвет основы**: Коричневый (RGB: 0.4, 0.35, 0.3)
- **Цвет краёв**: Тёмно-коричневый (RGB: 0.25, 0.2, 0.15)
- **Узор**: Имитация кирпичной кладки (6 блоков)
- **Полоска здоровья**: Аналогична монстрам (зелёная/красная, 4px)

### Геймплей
- **Позиция**: Только в зоне врагов (0..ENEMY_ROWS-1)
- **Статичность**: Не двигаются, всегда остаются на месте
- **Здоровье**: Настраивается в конфиге уровня (обычно 2-5 HP)
- **Разрушение**: При HP = 0 исчезают с поля

## Функциональность

### 1. Блокировка монстров

Монстры не могут пройти через препятствия и вынуждены их обходить:

#### Движение вперёд
```gdscript
if y + 1 < ENEMY_ROWS:
    var has_obstacle = obstacles[y+1][x] > 0
    if not occupied_next[y+1][x] and not has_obstacle:
        # Движение разрешено
```

#### Движение в сторону
```gdscript
var has_obstacle = obstacles[y][nx] > 0
if not occupied_next[y][nx] and not has_obstacle:
    # Движение в сторону разрешено
```

### 2. Взаимодействие со снарядами

Снаряды попадают в препятствия и наносят урон:

```gdscript
# Проверяем препятствие первым (приоритет перед монстрами)
if obstacles[ty][tx] > 0:
    obstacles[ty][tx] -= 1
    if obstacles[ty][tx] <= 0:
        # Препятствие разрушено
        obstacles[ty][tx] = 0
        obstacles_initial_hp[ty][tx] = 0
```

### 3. Планирование целей

При выпуске снарядов препятствия имеют приоритет:

```gdscript
if obstacles[yy][col_x] > 0:
    # Препятствие блокирует — снаряд попадёт в него
    row_hp = obstacles[yy][col_x]
elif enemies[yy][col_x] > 0:
    # Нет препятствия — цель монстр
    row_hp = enemies[yy][col_x]
```

### 4. Спавн монстров

Новые монстры не появляются на клетках с препятствиями:

```gdscript
# При начальном заполнении
if obstacles[y][x] > 0:
    continue  # Пропускаем клетку с препятствием

# При спавне в верхнем ряду
if enemies[0][x] == 0 and obstacles[0][x] == 0:
    # Можно спавнить монстра
```

## Технические детали

### Структуры данных

```gdscript
var obstacles := []              # 2D массив HP препятствий [y][x]
var obstacles_initial_hp := []   # 2D массив начального HP [y][x]
```

### Константы

```gdscript
const OBSTACLE_COLOR := Color(0.4, 0.35, 0.3, 1.0)          # Основной цвет
const OBSTACLE_EDGE_COLOR := Color(0.25, 0.2, 0.15, 1.0)   # Цвет краёв
```

### Инициализация из конфига

```gdscript
func _init_obstacles_from_config(cfg: Dictionary):
    obstacles.clear()
    obstacles_initial_hp.clear()
    
    # Создаём пустую сетку
    for y in range(ENEMY_ROWS):
        var row := []
        var row_init := []
        for x in range(COLS):
            row.append(0)
            row_init.append(0)
        obstacles.append(row)
        obstacles_initial_hp.append(row_init)
    
    # Загружаем препятствия из конфига
    if cfg.has("obstacles") and typeof(cfg.obstacles) == TYPE_ARRAY:
        for obs in cfg.obstacles:
            if typeof(obs) == TYPE_DICTIONARY:
                var ox = int(obs.x)
                var oy = int(obs.y)
                var hp = int(obs.hp)
                if oy >= 0 and oy < ENEMY_ROWS and ox >= 0 and ox < COLS:
                    obstacles[oy][ox] = hp
                    obstacles_initial_hp[oy][ox] = hp
```

### Отрисовка

```gdscript
func _draw_obstacle(top_left: Vector2, size: Vector2, hp: int, max_hp: int):
    # 1. Тень
    draw_rect(Rect2(top_left + Vector2(2, 2), size - Vector2(4, 4)), Color(0, 0, 0, 0.3))
    
    # 2. Основа
    draw_rect(Rect2(top_left, size), OBSTACLE_COLOR)
    
    # 3. Обводка
    draw_line(top_left, top_left + Vector2(size.x, 0), OBSTACLE_EDGE_COLOR, 3.0)
    draw_line(top_left, top_left + Vector2(0, size.y), OBSTACLE_EDGE_COLOR, 3.0)
    draw_line(top_left + Vector2(size.x, 0), top_left + size, OBSTACLE_EDGE_COLOR, 3.0)
    draw_line(top_left + Vector2(0, size.y), top_left + size, OBSTACLE_EDGE_COLOR, 3.0)
    
    # 4. Узор кирпичной кладки (6 блоков)
    var block_pattern = [
        [0.0, 0.0, 0.5, 0.33],    # Верхний левый
        [0.5, 0.0, 1.0, 0.33],    # Верхний правый
        [0.0, 0.33, 0.4, 0.66],   # Средний левый
        [0.4, 0.33, 1.0, 0.66],   # Средний правый
        [0.0, 0.66, 0.6, 1.0],    # Нижний левый
        [0.6, 0.66, 1.0, 1.0]     # Нижний правый
    ]
    
    for block in block_pattern:
        var bx = top_left.x + size.x * block[0]
        var by = top_left.y + size.y * block[1]
        var bw = size.x * (block[2] - block[0])
        var bh = size.y * (block[3] - block[1])
        draw_rect(Rect2(bx, by, bw, bh), OBSTACLE_EDGE_COLOR, false, 1.5)
    
    # 5. Полоска здоровья
    _draw_monster_health_bar(top_left, size.x, hp, max_hp, 1.0)
```

## Конфигурация уровня

### Формат JSON

```json
{
  "cols": 8,
  "rows": 16,
  "enemy_rows": 10,
  "moves": 15,
  "monster_tiers": [
    { "hp": 1, "count": 12 },
    { "hp": 2, "count": 8 }
  ],
  "obstacles": [
    { "x": 2, "y": 3, "hp": 3 },
    { "x": 5, "y": 3, "hp": 3 },
    { "x": 3, "y": 5, "hp": 5 }
  ]
}
```

### Поля препятствия

- **x** (int): Колонка (0..COLS-1)
- **y** (int): Ряд (0..ENEMY_ROWS-1)
- **hp** (int): Начальное здоровье (обычно 2-5)

### Валидация

- `x` должен быть в диапазоне [0, COLS-1]
- `y` должен быть в диапазоне [0, ENEMY_ROWS-1]
- `hp` должен быть > 0
- Препятствия не должны перекрываться

## Стратегические применения

### 1. Защита сильных монстров

Разместить препятствие перед колонкой с сильным монстром:

```json
{
  "obstacles": [
    { "x": 3, "y": 5, "hp": 5 }
  ],
  "monster_tiers": [
    { "hp": 5, "count": 1 }  // В колонке 3
  ]
}
```

### 2. Создание лабиринта

Расставить препятствия так, чтобы монстры вынуждены были двигаться по определённому пути:

```json
{
  "obstacles": [
    { "x": 2, "y": 3, "hp": 3 },
    { "x": 5, "y": 3, "hp": 3 },
    { "x": 3, "y": 5, "hp": 4 },
    { "x": 4, "y": 5, "hp": 4 }
  ]
}
```

### 3. Блокировка центральных колонок

Заставить монстров распределяться по краям поля:

```json
{
  "obstacles": [
    { "x": 3, "y": 2, "hp": 5 },
    { "x": 4, "y": 2, "hp": 5 }
  ]
}
```

## Игровая механика

### Приоритеты

1. **Снаряды**: Препятствие > Монстр
2. **Движение монстров**: Обход препятствий
3. **Спавн**: Не на препятствиях

### Поведение при разрушении

```gdscript
if obstacles[ty][tx] <= 0:
    obstacles[ty][tx] = 0
    obstacles_initial_hp[ty][tx] = 0
    # VFX разрушения
    _board_vfx.append({
        "type": "explosion",
        "pos": obs_center,
        "color": OBSTACLE_COLOR,
        "t": 0.0,
        "d": 0.3
    })
```

## Ограничения и особенности

### Текущие ограничения

1. Препятствия не двигаются (статичные)
2. Нет анимации разрушения (кроме VFX взрыва)
3. Один тип препятствия (визуально)
4. Препятствия только в зоне врагов

### Возможные улучшения

1. **Разные типы препятствий**:
   - Деревянные (мало HP, быстро разрушаются)
   - Каменные (много HP, медленно разрушаются)
   - Железные (очень много HP)

2. **Визуальные улучшения**:
   - Текстуры вместо прямоугольников
   - Анимация разрушения (трещины, обломки)
   - Эффекты частиц при разрушении

3. **Геймплейные улучшения**:
   - Препятствия, которые регенерируют HP
   - Препятствия, которые взрываются при разрушении
   - Препятствия-ловушки для монстров

4. **Интерактивность**:
   - Препятствия, которые можно двигать
   - Препятствия, которые меняют направление снарядов
   - Препятствия-порталы

