# Changelog - Полоска здоровья монстров

## [2024-03-21] - Реализация полоски здоровья

### Добавлено

#### Функционал
- Компактная полоска здоровья над монстрами (высота 4px)
- Визуальная индикация текущего HP (зелёный цвет)
- Визуальная индикация полученного урона (красный цвет)
- Поддержка прозрачности для анимаций
- Синхронизация с idle-анимациями монстров (дыхание, покачивание)
- Синхронизация с анимациями получения урона (shake)
- Автоматическое исчезновение при уничтожении монстра

#### Константы
```gdscript
HEALTH_BAR_HEIGHT := 4.0                              # Высота полоски
HEALTH_BAR_MARGIN := 2.0                              # Отступ от монстра
HEALTH_BAR_BG_COLOR := Color(0.2, 0.2, 0.2, 0.6)      # Цвет фона
HEALTH_BAR_HEALTH_COLOR := Color(0.2, 0.8, 0.2, 0.9)  # Цвет здоровья
HEALTH_BAR_DAMAGE_COLOR := Color(0.9, 0.2, 0.2, 0.7)  # Цвет урона
```

### Технические детали

#### Алгоритм отрисовки
1. Проверка валидности `max_hp > 0`
2. Расчёт позиции полоски `(top_left.y - HEALTH_BAR_HEIGHT - HEALTH_BAR_MARGIN)`
3. Расчёт соотношения здоровья `health_ratio = hp / max_hp`
4. Отрисовка фона (тёмно-серый)
5. Отрисовка зелёной части (текущее HP)
6. Отрисовка красной части (полученный урон), если `hp < max_hp`

#### Позиционирование
- Полоска следует за `anim_top_left` монстра
- Учитывается `shake_offset` при тряске
- Ширина полоски = `anim_size.x` (ширина монстра)

#### Производительность
- Минимальные вычисления (только арифметика)
- Простые прямоугольники (draw_rect)
- Нет создания новых объектов
- Нет сложных эффектов

### Известные ограничения

- Нет анимации плавного перехода при изменении HP
- Нет численного отображения HP
- Отсутствует поддержка препятствий (пока не реализованы в игре)

### Будущие улучшения

1. **Анимация изменения HP**
   - Плавный переход от зелёного к красному
   - Эффект "мерцания" при попадании

2. **Численные индикаторы** (опционально)
   - Отображение "3/5" над полоской
   - Компактный шрифт

3. **Визуальные эффекты**
   - Пульсация при критическом HP (< 20%)
   - Градиенты вместо однородной заливки
   - Светлая обводка для контраста

4. **Адаптивность**
   - Изменение цвета в зависимости от % HP
   - Специальные индикаторы для боссов

5. **Препятствия**
   - Поддержка полосок для препятствий
   - Различные стили для разных типов объектов
