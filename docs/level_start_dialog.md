# Диалог старта уровня

## Описание

Диалоговое окно, которое появляется перед началом каждого уровня. Показывает игроку текущий прогресс Шлема Морта, позволяет выбрать предуровневые усиления и запустить уровень.

```
┌─────────────────────────────────────┐
│  [✕]                    ← Крестик   │
│                                     │
│         УРОВЕНЬ 7                   │
│                                     │
│       ШЛЕМ МОРТА                    │
│      [ 1 ] [ 2 ] [ 3 ]              │
│   Бонус: 2 стрелы + 2 бомбы         │
│                                     │

│   [🧨]   [➡️]   [🌈]                │
│   Бомба  Стрела  Шар                │
│    x2     x3     x1                 │
│   [ + ]                             │
│                                     │
│        [ ИГРАТЬ ]                   │
└─────────────────────────────────────┘
```

## Логика работы

### Инициализация

1. Диалог создается в `game_board.gd` при вызове `_show_level_start_dialog()`
2. Загружается скрипт `level_start_dialog.gd`
3. Вызывается метод `setup()` для построения UI
4. Блокируется игровой ввод (`set_process_unhandled_input(false)`)

### Взаимодействие с игроком

1. **Выбор усиления**:
   - Игрок кликает на кнопку усиления
   - Вызывается `LevelManager.use_prelevel_boost(type)`
   - Счетчик уменьшается на 1
   - Кнопка переходит в состояние "Выбрана"

2. **Отмена выбора**:
   - Игрок повторно кликает на выбранную кнопку
   - Заряд возвращается обратно (`prelevel_boosts[type] += 1`)
   - Кнопка возвращается в обычное состояние

3. **Покупка зарядов**:
   - Игрок кликает на кнопку "+"
   - Вызывается `_show_buy_prelevel_boost_dialog(type)`
   - TODO: Показывается диалог покупки с проверкой монет

4. **Закрытие диалога**:
   - Игрок кликает на крестик или кнопку "Играть"
   - Вызывается `_on_start_pressed()`
   - Испускается сигнал `start_gameplay` с выбранными усилениями
   - Диалог удаляется (`queue_free()`)

### Запуск уровня

После закрытия диалога в `game_board.gd`:
1. Обработчик `_on_level_start_dialog_completed()` получает выбранные усиления
2. Сохраняются в переменные `_selected_prelevel_boosts` и `_mort_helmet_bonus_chips`
3. Вызывается `_spawn_bonus_chips_at_start()` для размещения бонусных фишек
4. Разблокируется игровой ввод (`set_process_unhandled_input(true)`)
5. Игра начинается

var _selected_prelevel_boosts := {

}

## Примеры использования

### Сценарий 1: Новый игрок, нет win streak

- **Шлем Морта**: Уровень 0, все индикаторы серые
- **Текст**: "Победите уровни подряд для усиления!"
- **Усиления**: x3 каждого типа (начальный набор)
- **Действие**: Выбирает бомбу, кликает "Играть"
- **Результат**: На поле появляется 1 бомба в случайном месте

### Сценарий 2: Игрок с win streak = 2

- **Шлем Морта**: Уровень 2, индикаторы 1 и 2 золотые
- **Текст**: "Бонус: 2 стрелы + 2 бомбы"
- **Усиления**: x1 бомб, x0 стрел, x2 шаров
- **Действие**: Кликает "+", покупает стрелы, выбирает 1 стрелу и 1 шар
- **Результат**: На поле появляется 4 стрелы (2 от шлема + 1 выбрана + 1 дубль?) и 3 бомбы (2 от шлема + 1 шар становится бомбой?)
- **Исправление**: 2 стрелы (шлем) + 1 стрела (выбор) + 2 бомбы (шлем) + 1 шар (выбор) = 6 бонусных фишек

### Сценарий 3: У игрока закончились все усиления

- **Усиления**: x0 всех типов
- **Отображение**: Все кнопки серые и disabled, под каждой кнопка "+"
- **Действие**: Может купить нужные усиления за 200 монет (по 3 заряда)
- **Альтернатива**: Играет без предуровневых усилений (только с бонусами шлема)

## UX соображения

### Преимущества дизайна

1. **Информативность**: Игрок сразу видит свой прогресс и доступные опции
2. **Стратегия**: Можно планировать использование усилений на сложных уровнях
3. **Визуальная награда**: Золотые индикаторы шлема показывают достижения
4. **Гибкость**: Можно начать без усилений или с максимальным набором

### Возможные улучшения

1. **Анимация появления**: Плавное fade-in вместо мгновенного появления
2. **Превью бонусных фишек**: Показать, где они появятся на поле
3. **Статистика**: Отображение процента побед с текущим уровнем шлема
4. **Подсказки**: Tooltips при наведении на иконки усилений
5. **Звук**: Звуковые эффекты при выборе и покупке усилений
