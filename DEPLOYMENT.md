# 🚀 Автоматическая сборка и деплой Godot Web

## Что настроено

Этот проект настроен для автоматической сборки веб-версии игры через GitHub Actions. Каждый раз, когда вы делаете `push` в ветки `main` или `cursor/godot-web-9125`, автоматически:

1. ✅ Скачивается Godot Engine 4.6
2. ✅ Экспортируется веб-версия игры
3. ✅ Публикуется на GitHub Pages
4. ✅ Становится доступна для тестирования на телефоне

## 📱 Как включить GitHub Pages

После первого push вам нужно **один раз** включить GitHub Pages в настройках репозитория:

1. Откройте ваш репозиторий на GitHub
2. Перейдите в **Settings** (Настройки)
3. В левом меню найдите **Pages**
4. В разделе **Source** (Источник):
   - Branch: выберите `gh-pages`
   - Folder: выберите `/ (root)`
5. Нажмите **Save**

## 🌐 Как открыть игру на телефоне

После того как GitHub Actions завершит сборку (обычно 3-5 минут), ваша игра будет доступна по адресу:

```
https://<ваш-username>.github.io/<название-репозитория>/
```

Например, если ваш username `ivanov` и репозиторий называется `elemental-blast`, то адрес будет:
```
https://ivanov.github.io/elemental-blast/
```

Просто откройте этот адрес в браузере на телефоне!

## 🔍 Проверка статуса сборки

1. Перейдите в раздел **Actions** в вашем репозитории на GitHub
2. Там вы увидите все запуски workflow "Godot Web Export"
3. Зелёная галочка ✅ = сборка успешна
4. Красный крестик ❌ = ошибка (кликните, чтобы посмотреть логи)

## 🛠️ Как это работает

### Автоматический запуск

Сборка запускается автоматически при:
- Push в ветку `main`
- Push в ветку `cursor/godot-web-9125`
- Создании Pull Request в `main`

### Процесс сборки

1. **Checkout** - скачивается код репозитория
2. **Setup Godot** - устанавливается Godot Engine и шаблоны экспорта
3. **Setup Export Preset** - создаётся конфигурация для веб-экспорта
4. **Build** - собирается веб-версия игры
5. **Add Headers** - добавляются необходимые HTTP-заголовки для работы в браузере
6. **Optimize for Mobile** - создаётся оптимизированная HTML-страница для мобильных устройств
7. **Deploy** - публикация на GitHub Pages

### Мобильная оптимизация

HTML-страница включает:
- ✅ Адаптивный viewport для мобильных устройств
- ✅ Полноэкранный режим
- ✅ Отключение масштабирования пальцами (для игр)
- ✅ Индикатор загрузки
- ✅ Обработка ошибок

## 🎮 Разработка

Теперь ваш workflow:

1. **Вносите изменения** в Godot или редакторе кода
2. **Commit & Push** в ветку `cursor/godot-web-9125`
   ```bash
   git add .
   git commit -m "Добавил новый уровень"
   git push
   ```
3. **Подождите 3-5 минут** пока GitHub Actions соберёт проект
4. **Откройте на телефоне** ссылку на GitHub Pages
5. **Тестируйте** изменения прямо в браузере!

## 🐛 Решение проблем

### Игра не загружается на телефоне

1. Проверьте, что GitHub Pages включён (см. раздел выше)
2. Проверьте, что сборка прошла успешно в разделе Actions
3. Попробуйте открыть в режиме инкогнито (может быть кэш)
4. Проверьте консоль браузера на ошибки (на Android: Chrome DevTools через USB)

### Сборка завершается с ошибкой

1. Откройте раздел **Actions** на GitHub
2. Кликните на неудачный запуск
3. Посмотрите логи - обычно ошибка видна в выводе
4. Частые причины:
   - Ошибки в сценах Godot (некорректные ссылки)
   - Отсутствующие ресурсы
   - Проблемы с экспортом

### Старая версия на GitHub Pages

GitHub Pages может кэшировать содержимое. Попробуйте:
- Очистить кэш браузера
- Открыть в режиме инкогнито
- Подождать 5-10 минут
- Добавить `?v=123` к URL (меняйте число каждый раз)

## 📝 Дополнительная настройка

### Изменить версию Godot

Откройте файл `.github/workflows/godot-web-export.yml` и измените:
```yaml
env:
  GODOT_VERSION: 4.6  # <- измените на нужную версию
```

### Собирать только из main

Уберите `cursor/godot-web-9125` из триггеров в workflow файле:
```yaml
on:
  push:
    branches:
      - main
      # - cursor/godot-web-9125  # <- закомментируйте
```

### Добавить другие ветки

Добавьте нужные ветки в список:
```yaml
on:
  push:
    branches:
      - main
      - cursor/godot-web-9125
      - develop  # <- добавьте свою ветку
```

## 📦 Артефакты сборки

Каждая сборка сохраняет веб-версию как артефакт, который можно скачать вручную:

1. Перейдите в **Actions**
2. Выберите успешный запуск
3. Внизу страницы найдите **Artifacts**
4. Скачайте **web-build**
5. Распакуйте и откройте `index.html` локально

Артефакты хранятся 14 дней.
